# Life RPG — MVP с GitHub Gist Sync

Одностраничный RPG-трекер реальной жизни.

## Что изменилось в этой версии

- Добавлена синхронизация сохранения через GitHub Gist.
- Добавлены кнопки:
  - `Загрузить из Gist`
  - `Сохранить в Gist`
  - `Создать secret Gist`
- Разделы страницы стали сворачиваемыми.
- Добавлены маленькие PNG-иконки для быстрой идентификации разделов.
- Сохранена логика `навыки + метрики`:
  - навыки дают XP;
  - метрики не дают прямой XP, но закрывают дейлики;
  - отжимания и подтягивания одновременно являются навыками и дневными метриками.

## Как запустить локально

Открой `index.html` в браузере.

Если браузер блокирует запросы к GitHub API при открытии файла напрямую, запусти локальный сервер:

```bash
python -m http.server 8000
```

Потом открой:

```text
http://localhost:8000
```

## Как выложить на GitHub Pages

1. Создай репозиторий.
2. Загрузи файлы:
   - `index.html`
   - `styles.css`
   - `app.js`
   - папку `icons`
   - `README.md`
3. В настройках репозитория включи GitHub Pages из ветки `main`.

## Как подключить GitHub Gist Sync

### 1. Создай GitHub token

Рекомендуемый способ для личного MVP:

1. Открой GitHub.
2. Перейди в `Settings`.
3. Открой `Developer settings`.
4. Открой `Personal access tokens`.
5. Создай token с минимальными правами для Gist.

Для classic token нужен scope:

```text
gist
```

Важно: не давай токену доступ к репозиториям, если он нужен только для сохранения Life RPG.

### 2. Открой Life RPG

Открой раздел `Облако GitHub Gist`.

Заполни поля:

- `GitHub token` — твой token.
- `Имя файла в Gist` — можно оставить `life-rpg-save.json`.
- `Gist ID` — можно оставить пустым, если создаёшь новый Gist из приложения.

### 3. Создай secret Gist

Нажми:

```text
Создать secret Gist
```

Приложение создаст Gist и само подставит `Gist ID`.

### 4. Как пользоваться с телефона и компьютера

Правило простое:

1. Перед работой на устройстве нажми `Загрузить из Gist`.
2. Внеси активности.
3. После изменений нажми `Сохранить в Gist`.

Так ты не будешь случайно работать со старой локальной копией.

## Безопасность

В этой версии token вводится в браузере.

Есть два режима:

- если галочка `Хранить token в этом браузере` выключена — token не сохраняется после обновления страницы;
- если включена — token хранится в `localStorage` конкретного браузера.

Для личного MVP это приемлемо, но важно:

- не публикуй token;
- не вставляй token в `app.js`;
- не коммить token в GitHub;
- если token утёк — сразу отзови его в настройках GitHub.

## Текущая модель данных

### Навыки

Навыки дают XP и качают характеристики:

- Плавание
- Отжимания
- Подтягивания
- Жим / силовая
- Мобильность / ТБС
- Сон
- Белок

### Метрики

Метрики не дают прямого XP, но закрывают дейлики и цели:

- Шаги
- Дистанция ходьбы
- Активные ккал
- Вес

Отжимания, подтягивания, сон, белок и мобильность тоже считаются дневными метриками, потому что по ним могут закрываться дейлики.

## Баланс дейликов

Стандартные дейлики дают фиксированный XP.

Кастомные дейлики дают процентный бонус от XP за соответствующий навык за день.

Рекомендации:

- лёгкая цель: 5%
- нормальная цель: 10%
- тяжёлая цель: 15–20%
- выше 25% лучше не ставить, чтобы не ломать прогрессию

## Экспорт и импорт

Даже с Gist Sync лучше иногда делать локальный backup:

- `Экспорт JSON` — скачать сохранение файлом;
- `Импорт JSON` — загрузить сохранение из файла.
